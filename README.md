MNIST-GAN-LightningA professional, modular implementation of a Generative Adversarial Network (GAN) using PyTorch Lightning. This project demonstrates how to stabilize GAN training using manual optimization, deep convolutional layers, and automated data pipelines.📌 Project OverviewThis repository implements a DCGAN-inspired architecture to generate handwritten digits.The Generator transforms latent noise into $28 \times 28$ grayscale images using Transposed Convolutions.The Discriminator acts as a binary classifier, distinguishing real MNIST samples from generated fakes.🚀 Key FeaturesManual Optimization: Full control over the Minimax game between the Generator and Discriminator.Stable Architectures: Uses Batch Normalization and LeakyReLU to prevent mode collapse.Mixed Precision: Supports 16-mixed precision for faster training on modern GPUs.Automated Logging: Integrated with TensorBoard to track g_loss and d_loss in real-time.🛠️ Setup & InstallationClone the repo:Bashgit clone https://github.com/your-username/mnist-gan-lightning.git
cd mnist-gan-lightning
Install dependencies:Bashpip install -r requirements.txt
Train the model:Bashpython train.py
🏗️ Modular Architecture & ScalabilityThis project follows the Open-Closed Principle. The code is decoupled so you can iterate on components without touching the core training logic:Swap Data: Simply create a Cifar10DataModule in src/datamodule.py and pass it to the trainer in train.py.Iterate Models: You can drop in a GeneratorV2 (e.g., using Self-Attention or Residual blocks) into src/models.py. As long as the input/output shapes remain consistent, the train.py flow remains unbroken.Hardware Agnostic: Thanks to the accelerator="auto" configuration, the same code runs seamlessly on CPU, MPS (Mac), or CUDA (NVIDIA).📊 ResultsAfter 20 epochs, the Generator produces clear, diverse handwritten digits from pure random noise. Check the lightning_logs/ folder to view progress in TensorBoard:Bashtensorboard --logdir lightning_logs/
